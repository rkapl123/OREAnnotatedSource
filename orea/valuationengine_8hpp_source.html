41 //! Valuation Engine
42 /*!
43  The valuation engine's purpose is to generate an NPV cube.
44  Its buildCube loops over samples->dates->trades and prices the portfolio
45  under all samples and dates.
46 
47  The number of dates is defined by the DateGrid passed to the constructor.
48  The number of trades is defined by the size of the portfolio passed to buildCube().
49  The number of samples is defined by the NPVCube that is passed to buildCube(), this
50  can be dynamic.
51 
52  In addition to storing the resulting NPVs it can be given any number of calculators
53  that can store additional values in the cube.
54 
55  \ingroup simulation
56 */
57 class ValuationEngine : public ore::data::ProgressReporter {
