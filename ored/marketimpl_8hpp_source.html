183  /*! Require a market object, this can be used in derived classes to build objects lazily. If the
184  method is not overwritten in a derived class, it is assumed that the class builds all market
185  object upfront.
186 
187  For FXVols and Correlations the require is not 'hard', e.g. both EURUSD and USDEUR might
188  be required for FXVols, but only one of them is expected to be actually built (the other
189  one is then constructed on the fly from the first one). Therefore no error should be thrown
190  in the implementation of require(), if an object is ultimately not found, an appropriate error
191  will be thrown from this class.
192 
193  An object is required for a single configuration. If it can't be built for this configuration,
194  it should be tried to be built for the "default" configuration instead, because this is used
195  as a fallback.
196 
197  Notice that correlation curves are required with '&' as a delimiter between the indexes. */
198  virtual void require(const MarketObject o, const string& name, const string& configuration,
199  const bool forceBuild = false) const {}
