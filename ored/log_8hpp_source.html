197 //! Global static Log class
198 /*!
199  The Global Log class gets registered with individual loggers and receives application log messages.
200  Once a message is received, it is immediately dispatched to each of the registered loggers, the order in which
201  the loggers are called is not guaranteed.
202 
203  Logging is done by the calling thread and the LOG call blocks until all the loggers have returned.
204 
205  At start up, the Log class has no loggers and so will ignore any LOG() messages until it is configured.
206 
207  To configure the Log class to log to a file "/tmp/my_log.txt"
208  <pre>
209  Log::instance().removeAllLoggers();
210  Log::instance().registerLogger(boost::shared_ptr<Logger>(new FileLogger("/tmp/my_log.txt")));
211  </pre>
212 
213  To change the Log class to only use a BufferLogger the user must call
214  <pre>
215  Log::instance().removeAllLoggers();
216  Log::instance().registerLogger(boost::shared_ptr<Logger>(new BufferLogger));
217  </pre>
218  and then to retrieve log messages from the buffer and print them to stdout the user must call:
219  <pre>
220  std::cout << "Begin Log Messages:" << std::endl;
221 
222  boost::shared_ptr<BufferLogger> bl = boost::dynamic_pointer_cast<BufferLogger>
223  (Log::instance().logger(BufferLogger::name));
224 
225  while (bl.hasNext())
226  std::cout << bl.next() << std::endl;
227  std::cout << "End Log Messages." << std::endl;
228  </pre>
229  \ingroup utilities
230  */

360 //! LoggerStream class that is a std::ostream replacement that will log each line
361 /*! LoggerStream is a simple wrapper around a string stream, it has an explicit
362  cast std::ostream& method so it can be used in place of any std::ostream, this
363  can be used with QuantExt methods that take a std::ostream& for logging purposes.
364 
365  Once the stream falls out of focus, it's destructor will take the buffered log
366  messages and pass them the main ore::data::Log::instance().
367 
368  Note the following
369  - The timestamps for each log message will correspond to when the LoggerStream
370  destructor has been called, this may not correspond to the actual time the event
371  occurred.
372  - The filename and linenumber in the ore::data::Log() have to be explicitly passed to
373  the LoggerStream, as such the log messages will not correspond to any references
374  in QuantExt (or any other library).
375 
376  The LoggerStream relies on falling out of scope to trigger the logging, as such they
377  should not be created using the new operator.
378 
379  The proper usage is to use the macro LOGGERSTREAM and DLOGGERSTREAM, if a function takes
380  a std::ostream& as a parameter, use the macro instead.
381 
382  \code{.cpp}
383  void function(int x, int y, std::ostream& out);
384 
385  void main () {
386 
387  // call function
388  function (3, 4, LOGGERSTREAM);
389  // All logging will be completed before this line
390  }
391  \endcode
392  */

394 public:
395  LoggerStream(unsigned mask, const char* filename, unsigned lineNo);
