199//! Global static Log class
200/*!
201 The Global Log class gets registered with individual loggers and receives application log messages.
202 Once a message is received, it is immediately dispatched to each of the registered loggers, the order in which
203 the loggers are called is not guaranteed.
204
205 Logging is done by the calling thread and the LOG call blocks until all the loggers have returned.
206
207 At start up, the Log class has no loggers and so will ignore any LOG() messages until it is configured.
208
209 To configure the Log class to log to a file "/tmp/my_log.txt"
210 <pre>
211 Log::instance().removeAllLoggers();
212 Log::instance().registerLogger(boost::shared_ptr<Logger>(new FileLogger("/tmp/my_log.txt")));
213 </pre>
214
215 To change the Log class to only use a BufferLogger the user must call
216 <pre>
217 Log::instance().removeAllLoggers();
218 Log::instance().registerLogger(boost::shared_ptr<Logger>(new BufferLogger));
219 </pre>
220 and then to retrieve log messages from the buffer and print them to stdout the user must call:
221 <pre>
222 std::cout << "Begin Log Messages:" << std::endl;
223
224 boost::shared_ptr<BufferLogger> bl = boost::dynamic_pointer_cast<BufferLogger>
225 (Log::instance().logger(BufferLogger::name));
226
227 while (bl.hasNext())
228 std::cout << bl.next() << std::endl;
229 std::cout << "End Log Messages." << std::endl;
230 </pre>
231 \ingroup utilities
232 */

386//! LoggerStream class that is a std::ostream replacement that will log each line
387/*! LoggerStream is a simple wrapper around a string stream, it has an explicit
388 cast std::ostream& method so it can be used in place of any std::ostream, this
389 can be used with QuantExt methods that take a std::ostream& for logging purposes.
390
391 Once the stream falls out of focus, it's destructor will take the buffered log
392 messages and pass them the main ore::data::Log::instance().
393
394 Note the following
395 - The timestamps for each log message will correspond to when the LoggerStream
396 destructor has been called, this may not correspond to the actual time the event
397 occurred.
398 - The filename and linenumber in the ore::data::Log() have to be explicitly passed to
399 the LoggerStream, as such the log messages will not correspond to any references
400 in QuantExt (or any other library).
401
402 The LoggerStream relies on falling out of scope to trigger the logging, as such they
403 should not be created using the new operator.
404
405 The proper usage is to use the macro LOGGERSTREAM and DLOGGERSTREAM, if a function takes
406 a std::ostream& as a parameter, use the macro instead.
407
408 \code{.cpp}
409 void function(int x, int y, std::ostream& out);
410
411 void main () {
412
413 // call function
414 function (3, 4, LOGGERSTREAM);
415 // All logging will be completed before this line
416 }
417 \endcode
418 */
419class LoggerStream {
420public:
421 LoggerStream(unsigned mask, const char* filename, unsigned lineNo);
422
423 //! destructor - this is when the logging takes place.
424 ~LoggerStream();
425
426 //! cast this LoggerStream as a ostream&
427 operator std::ostream &() { return ss_; }
428
429private:
430 unsigned mask_;
431 const char* filename_;
432 unsigned lineNo_;
433 std::stringstream ss_;
434};
