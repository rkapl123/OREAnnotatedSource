87/*! - helper class that takes and index name string and provides the index type and a parsed version of
88 the index with no market data attached
89 - commodity indices can be of the extended form for scripting, see parseScriptedCommodityIndex for details

138 bool operator>=(const IndexInfo& j) const { return !((*this) < j); }

162/*! This method tries to parse an commodity index name used in the scripting context
163
164 0) COMM-name
165 1) COMM-name-YYYY-MM-DD
166 2) COMM-name-YYYY-MM
167 ===
168 3) CMMM-name#N#D#Cal
169 4) COMM-name#N#D
170 5) COMM-name#N
171 ===
172 6) COMM-name!N
173
174 Here 0) - 2) are corresponding to the usual ORE conventions while 3) - 6) are specific to the scripting
175 module: Expressions of the form 3) - 5) are resolved to one of the forms 1) and 2) using a given commodity
176 future expiry calculator as follows:
177
178 3) COMM-name#N#D#Cal is resolved to the (N+1)th future with expiry greater than the given obsDate advanced by D
179 business days w.r.t. Calendar Cal, N >= 0
180 4) as 3), Cal is taken as the commodity index's fixing calendar
181 5) as 4), D is set to 0 if not given
182 6) COMM-name!N is resolved to the future with month / year equal to the obsDate and monthOffst = N, N >=0
183
184 Notice that the forms 1) and 2) can be parsed without an obsDate and a commodity future convention given. If no
185 convention is given, the fixing calendar in the index is set to the NullCalendar. In case a commodity future
186 convention is given for the name, the fixing calendar is set to the calendar from the convention.
187
188 TODO if the form is COMM-name-YYYY-MM, the day of month of the expiry date will be set to 01, consistently
189 with the ORE index parser, even if a convention is present, that would allow us to determine the correct
190 expiry date. Should we use that latter date in the returned index?
191
192 Forms 3) to 6) on the other hand require a commodity future convention in any case, and an obsDate.
193*/
194

198/*! This method tries to parse an inflation index name used in the scripting context
199
200 1) EUHICPXT
201 2) EUHICPXT#F
202 3) EUHICPXT#L
203
204 Here 1) is the original form used in ORE. This represents a non-interpolated index.
205 2,3) is the extended form including a flag indicating the interpolation F (flat, =1) or L (linear)

207 The function returns a ql inflation index accounting for the interpolation (but without ts attached),
208 and the ORE index name without the #F, #L suffix.
