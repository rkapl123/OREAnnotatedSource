36 //! Intelligent FX price repository
37 /*! FX Triangulation is an intelligent price repository that will attempt to calculate FX spot values
38  *
39  * As quotes for currency pairs are added to the repository they are stored in an internal map
40  * If the repository is asked for the FX spot price for a given pair it will attempt the following:
41  * 1) Look in the map for the pair
42  * 2) Look for the reverse quote (EURUSD -> USDEUR), if found it will return an inverse quote.
43  * 3) Look through the map and attempt to find a bridging pair (e.g EURUSD and EURJPY for USDJPY)
44  * and return the required composite quote.
45  *
46  * In cases (2) and (3) the constructed quote is then stored in the map so subsequent calls will hit (1).
47  *
48  * The constructed quotes all reference the original quotes which are added by the addQuote() method
49  * and so if these original quotes change in the future, the constructed quotes will reflect the new
50  * value
51  *
52  * Warning: The result of getQuote() can depend on previous calls to getQuote(), since newly added
53  * pairs in 3) are candidates for bridging pairs in future calls and the bridging only uses one
54  * intermediate currency. This state-dependent behaviour will be removed in a future release.
55  *
56  * \ingroup marketdata
57  */
58 class FXTriangulation {
59 public:
60  //! Default ctor, once built the repo is empty
61  FXTriangulation() {}
62 
63  //! Add a quote to the repo
64  void addQuote(const std::string& pair, const Handle<Quote>& spot);
65 
66  //! Get a quote from the repo, this will follow the algorithm described above
67  Handle<Quote> getQuote(const std::string&) const;
68 
69  //! Get all quotes currently stored in the triangulation
70  const std::vector<std::pair<std::string, Handle<Quote>>>& quotes() const { return map_; }
71 
72 private:
73  mutable std::vector<std::pair<std::string, Handle<Quote>>> map_;
74 };
